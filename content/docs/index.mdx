## Terminology

Before diving in, a few terms used throughout:

### Levels / Maps

The word "map" generally refers to what the average person calls a "level" and may be used interchangeably.

### Identifiers

An `Identifier` is anything that resembles or serves as a "name" — the name of an asset, a material layer, a Blueprint property, a variable, a folder, a data table row, etc.

### Cases

There are a few different ways you can case words when naming:

| Case | Example | Description |
| --- | --- | --- |
| **PascalCase** | `DesertEagle`, `StyleGuide` | Capitalize every word, remove all spaces. |
| **camelCase** | `desertEagle`, `styleGuide` | First letter lowercase, following words capitalized. |
| **Snake_case** | `desert_Eagle`, `Style_Guide` | Words separated by underscores. |

### Variables / Properties

The words "variable" and "property" are generally interchangeable. When used together in the same context:

- **Property** usually refers to a variable defined in a class (e.g., `bExploded` is a property of `BP_Barrel`). Implies accessing previously defined data.
- **Variable** usually refers to a function argument or local variable. Implies discussion about its definition and what it will hold.
